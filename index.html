Creative Cloud Solutions: Asset Browser Declassified

High-level Overview

The Creative Cloud SDK includes a library to easily implement Creative Cloud file browsing. We do this using the Asset Browser. The Asset Browser is used through the AdobeUXAssetBrowser class which provides a simple UI for browsing the user's files as well as selecting one for download.

In this tutorial we will create a simple minimalist UI for testing purposes, enable the Asset Browser, enable the ability to select a file from the Asset Browser, and then display that file in our simple minimalist UI.

Assumptions

Before we start we are going to go through a few assumptions about how prepared we are to integrate the Creative Cloud Asset Browser into our app. Make sure to check every box below before moving forward.
Have a basic understanding of Android Development
Created an Android project using the Blank Activity template
Purchased and created a Creative Cloud account
Completed the Creative Cloud Getting Started tutorial
Integrated the Creative Cloud SDK
Integrated your Client Id and Secret by implementing IAdobeAuthClientCredentials
Updated the Android Manifest
Instantiated the AdobeAuthSessionHelper
Enabled Creative Cloud login with the AdobeUXAuthManager

File browsing with AdobeUXAssetBrowser

The Asset Browser is a simple UI that allows the user to browse, select, and download their files saved in the Creative Cloud. It was designed for simple and smooth integration.

We use the Asset Browser by instantiating the AdobeUXAssetBrowser class. To do that we are going to add a constant to our activity's global variables, add a method to use the Asset Browser, and then have the listener attached to our button call that method.

The AdobeUXAssetBrowser class has a few methods. In the above example we used two of those methods, getSharedInstance() and popupFileBrowser()
The AdobeUXAssetBrowser.getSharedInstance() is a static method that returns an instance of the Asset Browser as a singleton.
popupFileBrowser(Activity activity, int requestCode) is a void method that launches the Asset Browser. It extends the startActivityForResult() method of the Android framework which will be important later when we start to retrieve files from the Asset Browser.

The AdobeUXAssetBrowser class also allows a degree of customizability by passing a AdobeUXAssetBrowserConfiguation object as a third parameter to the popupFileBrowser() method. The AdobeUXAssetBrowserConfiguration object allows for many of different options. We will be only using one, My Account, which allows the user to look at their account settings as well as gives the user the ability to log out of their Creative Cloud account. 
                           
                                //function to launch the Asset Browser

                                    //This API pops up the default File Browser as an Activity

                                    AdobeUXAssetBrowser sharedAssetBrowserInstance = AdobeUXAssetBrowser.getSharedInstance();

//start of change -- start of change -- start of change -- start of change
                                    //creates the AssetBrowser configuration object to add options to 
                                    AdobeUXAssetBrowserConfiguration browserConfiguration = new AdobeUXAssetBrowserConfiguration();

                                    //adds an enum of options constants to the options property of AssetBrowser configuration object, in this case ENABLE_MYACCOUNT_OPTION which will enable My Account
                                    browserConfiguration.options = EnumSet.of(AdobeUXAssetBrowserOption.ENABLE_MYACCOUNT_OPTION);

                                    //launches the AdobeUXAssetBrowser singleton with customized options, in this case enabling My Account

//end of change -- end of change -- end of change -- end of change
                                }

Retrieving selected files

We have a minimal UI, and a functioning Asset Browser, now we are going to retrieve the file that we select from the Asset Browser.

Earlier we learned that the popupFileBrowser() extends the startActivityForResult() Android framework method. This enables retrieving a file from the Asset Browser to be no different than retrieving data from another activity upon completion. We do it in our onActivityResult() method.

                                        AdobeUXAssetBrowser.ResultProvider assetBrowserResult = new AdobeUXAssetBrowser.ResultProvider(data);

                                        ArrayList mSelectedAssetsList = assetBrowserResult.getSelectionAssetArray();
                                        
                                    }

                                        //do nothing

//end of change -- end of change -- end of change -- end of change

Here we make use of the ResultProvider helper class which makes accessing the files selected by the user very simple. We also created an ArrayList of AdobeSelection objects.

Before speaking of AdobeSelection objects we must cover AdobeAsset objects. AdobeAsset is the base class of all objects presented in the Asset Browser, and all objects stored in the Adobe Creative Cloud.

AdobeSelection is the base class of all objects selected from the Asset Browser. The Asset Browser displays many different types of files, that a user can select and retrieve from their Creative Cloud account. Since they all share the common base class of AdobeSelection they can all be passed as an ArrayList of AdobeSelection objects. This allows us to select different types of files and retrieve them all at the same time.

                                //method that converts the passed AdobeAsset into an image and sets the passed imageView to that image
                                private void convertToImage(AdobeAsset asset, final ImageView localImageView) {
                                    if (asset instanceof AdobeAssetFile) {
                                        final AdobeAssetFile aFile = (AdobeAssetFile) asset;
                                        Log.d("AtachImage","Calling AtachImage>>>>>>>>>>>>>>>>>>>>>>>");
                                        aFile.getRenditionWithType(AdobeAssetFileRenditionType.ADOBE_ASSET_FILE_RENDITION_TYPE_PNG,
                                                //set the dimensions of the image to those of the passed imageview
                                                new AdobeAssetImageDimensions(localImageView.getWidth() , localImageView.getHeight()),

                                                //if everything succesfull then the AdobeAsset is converted to an image and the image view is set to that image
                                                new IAdobeGenericRequestCallback<byte[], AdobeAssetException>() {
                                                    @Override
                                                    public void onCompletion(byte[] data) {
                                                        InputStream inData = new ByteArrayInputStream(data);
                                                        Bitmap image = BitmapFactory.decodeStream(inData);
                                                        localImageView.setImageBitmap(image);
                                                        Log.d("AtachImage", "COMPLETED getRenditionWithType>>>>>>>>>>>>>>>>>>>>>>>");
                                                    }

                                                    @Override
                                                    public void onCancellation() {
                                                    //do nothing
                                                    }

                                                    //if there is an error then we change the image view to have a red background and an error is logged. 
                                                    @Override
                                                    public void onError(AdobeAssetException error) {
                                                        localImageView.setBackgroundColor(Color.parseColor("#ffff251c"));
                                                        Log.d("ERROR", "ERROR getRenditionWithType>>>>>>>>>>>>>>>>>>>>>>>");
                                                    }

                                                    @Override
                                                    public void onProgress(double progress) {
                                                    //do nothing
                                                    }
                                                });
                                    }
                                }
//end of change -- end of change -- end of change -- end of change
The convertToImage() method takes an AdobeAsset and an ImageView as its parameters, converts the AdobeAsset into a Bitmap, and then sets the passed ImageView to the converted Bitmap. If for some reason an error occurs during this proccess we set the ImageViews' background to red, to let us know that something has gone wrong.
